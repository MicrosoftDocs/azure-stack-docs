<h1><a id="user-content-introduction-to-azure-stack-virtual-machines" class="anchor" aria-hidden="true" href="#introduction-to-azure-stack-virtual-machines"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction to Azure Stack virtual machines</h1>
<p><em>Applies to: Azure Stack integrated systems and Azure Stack Development Kit</em></p>
<p>Azure Stack offers virtual machines (VMs) as one type of an on-demand, scalable computing resource. You can choose a VM when you need more control over the computing environment. This article provides details before you create your first VM.</p>
<p>An Azure Stack VM gives you the flexibility of virtualization without the need to manage clusters or individual machines. However, you still need to maintain the VM by performing tasks such as configuring, patching, and installing the software that runs on it.</p>
<p>You can use Azure Stack virtual machines in several ways. For example:</p>
<ul>
<li>
<p><strong>Development and test</strong><br>
Azure Stack VMs enable you to create a computer with a specific configuration required to code and test an application.</p>
</li>
<li>
<p><strong>Applications in the cloud</strong><br>
Because demand for your application can fluctuate, it might make economic sense to run it on a VM in Azure Stack. You pay for extra VMs when you need them and shut them down when you don't.</p>
</li>
<li>
<p><strong>Extended datacenter</strong><br>
Virtual machines in an Azure Stack virtual network can be connected to your organization's network or to Azure.</p>
</li>
</ul>
<p>The VMs that your application uses can scale up, or scale out, to whatever is required to meet your needs.</p>
<h2><a id="user-content-what-do-i-need-to-think-about-before-creating-a-vm" class="anchor" aria-hidden="true" href="#what-do-i-need-to-think-about-before-creating-a-vm"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>What do I need to think about before creating a VM</h2>
<p>There are always design considerations when you build out an application infrastructure in Azure Stack. These aspects of a VM are important to think about before you start creating your infrastructure:</p>
<ul>
<li>The names of your application resources.</li>
<li>The size of the VM.</li>
<li>The maximum number of VMs that can be created.</li>
<li>The operating system that the VM runs.</li>
<li>The configuration of the VM after it starts.</li>
<li>The related resources that the VM needs.</li>
</ul>
<h3><a id="user-content-naming" class="anchor" aria-hidden="true" href="#naming"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Naming</h3>
<p>A virtual machine has a name assigned to it and it has a computer name configured as part of the operating system. The name of a VM can be up to 15 characters.</p>
<p>If you use Azure Stack to create the operating system disk, the computer name and the virtual machine name are the same. If you upload and use your own image that contains a previously configured operating system and use it to create a virtual machine, the names may be different. When you upload your own image file, as a best practice, make sure the computer name in the operating system and the virtual machine name are the same.</p>
<h3><a id="user-content-vm-size" class="anchor" aria-hidden="true" href="#vm-size"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>VM size</h3>
<p>The size of the VM that you use is determined by the workload that you want to run. The size that you choose then determines factors such as processing power, memory, and storage capacity. Azure Stack offers a variety of sizes to support many types of uses.</p>
<h3><a id="user-content-vm-limits" class="anchor" aria-hidden="true" href="#vm-limits"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>VM limits</h3>
<p>Your subscription has default quota limits in place that can impact the deployment of VMs for your project. The current limit on a per subscription basis is 20 VMs per region.</p>

Marketplace images are identified by image publisher, offer, SKU, and version (typically the version is specified as <strong>latest</strong>.)</p>

<p>VM extensions give your VM additional capabilities through post deployment configuration and automated tasks.
These common tasks can be accomplished using extensions:</p>
<ul>
<li>
<p><strong>Run custom scripts</strong><br>
The Custom Script extension helps you to configure workloads on the VM by running your script when the VM is provisioned.</p>
</li>
<li>
<p><strong>Deploy and manage configurations</strong><br>
The PowerShell Desired State Configuration (DSC) extension helps you set up DSC on a VM to manage configurations and environments.</p>
</li>
<li>
<p><strong>Collect diagnostics data</strong><br>
The Azure Diagnostics extension helps you configure the VM to collect diagnostics data that can be used to monitor the health of your application.</p>
</li>
</ul>
